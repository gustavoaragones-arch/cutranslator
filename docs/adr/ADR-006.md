# ADR-006: Tripe Sub-Canonical Architecture

Status: Accepted
Date: 2026-06-09

## Context

The `tripe` canonical currently covers all bovine stomach chambers as
a single entity. Research across three countries has revealed that
multiple cultures treat specific stomach chambers as distinct
ingredients with separate names, preparation methods, and market
identities:

**Japan:**
- Mino (ミノ) — rumen
- Hachinosu (ハチノス) — honeycomb tripe (reticulum)
- Senmai (センマイ) — omasum
Each is a distinct yakiniku menu item. Diners order them separately.

**China:**
- Máodǔ (毛肚) — omasum, premium Sichuan hot pot ingredient
- Bǎiyè (百叶) — leaf tripe (omasum variant)
- Niúdǔ (牛肚) — generic stomach
Máodǔ in particular is treated as a prestige ingredient, not
interchangeable with generic tripe.

**South Korea:**
- Yang (양) — rumen/honeycomb tripe (generic)
- Makchang (막창) — fourth stomach (abomasum), especially in
  Daegu beef culture

Currently modeled as altNames within the tripe canonical entry
for each country. Source file carries a deferred comment
flagging this decision.

## Options Considered

**Option A — Keep single tripe canonical, altNames only**
All stomach chambers remain under `tripe`. Regional names
capture the distinction via altNames. No structural change.

Pros: Simple. No cascade updates to existing entries.
Consistent with how the dataset handles other anatomical
sub-distinctions (e.g. fore_shank / hind_shank were kept
separate but striploin variants were not).

Cons: Loses culinary precision for three major offal
cultures. Máodǔ and mino are not interchangeable — a user
asking "what is máodǔ" gets "tripe" which is technically
correct but informationally weak. The altNames field
can carry the distinction but the cut page cannot
differentiate preparation or tradition at the canonical level.

**Option B — Split into sub-canonicals**
Add: rumen, honeycomb_tripe, omasum, abomasum as separate
CanonicalId entries. Retire or demote `tripe` to a grouping
concept.

Pros: Maximum precision. Each chamber gets its own cut page,
SVG overlay, tradition links.

Cons: Four new canonicals, four new SVG overlays (internal
anatomy — harder to draw than surface cuts), cascade updates
to all existing tripe entries across 15 countries, all
tradition pages referencing tripe. Significant scope.

**Option C — Hybrid: keep tripe canonical, add sub-canonical
axis nodes**
Keep `tripe` as the primary canonical. Add lightweight
sub-canonical axis nodes (similar to zebu_hump_fat_axis_af)
for the three most culturally significant chambers:
maodu (CN), mino (JP), makchang (KR). Each axis node
maps_to: ["tripe"] and carries its own regional name
and tradition links without requiring SVG overlays.

Pros: Preserves existing data integrity. Adds precision
where cultures demand it without forcing a full canonical
split. No new SVG work. Tradition pages continue referencing
`tripe`. Axis nodes can link to the tripe cut page.

Cons: Adds architectural complexity. Axis nodes are
a Tier 3 concept that has only been used once (zebu hump).
Requires extending the axis node pattern to the offal
product.

## Decision

**Option C — Hybrid axis node approach.**

Rationale: The three chambers that demand distinction
(máodǔ, mino, makchang) are culturally significant enough
to deserve their own searchable identity but do not meet
the full canonical threshold — they are sub-cuts of a
single organ, not independently butchered primal regions.
The axis node pattern established for zebu_hump_fat_axis_af
provides the right precedent: a lightweight Tier 3 node
that adds cultural precision without structural disruption.

Full Option B remains available if future research reveals
that the chamber distinction is meaningful in enough
additional cultures to justify four new SVG overlays
and a full cascade update.

## Implementation

Three axis nodes to be added in a future batch:

1. maodu_axis_cn
   maps_to: ["tripe"]
   localName: "máodǔ", nativeScript: "毛肚"
   country: CN
   note: "Omasum. Premium Sichuan hot pot ingredient.
     Dipped briefly in boiling málà broth."

2. mino_axis_jp
   maps_to: ["tripe"]
   localName: "mino", nativeScript: "ミノ"
   country: JP
   note: "Rumen. Standard yakiniku cut.
     Grilled over charcoal."

3. makchang_axis_kr
   maps_to: ["tripe"]
   localName: "makchang", nativeScript: "막창"
   country: KR
   note: "Fourth stomach (abomasum).
     Especially associated with Daegu beef culture."

The deferred comments in data/offal/regionalNames.ts
for JP and KR tripe entries should be updated to
reference this ADR once axis nodes are implemented.

## Consequences

- tripe canonical remains unchanged
- Existing 15-country tripe entries unchanged
- Existing tradition pages unchanged
- Three axis nodes added in a dedicated batch (OX-4)
- Second country sweep proceeds without tripe
  sub-canonical ambiguity
- If future research adds 3+ more cultures with
  distinct chamber terminology, Option B is revisited

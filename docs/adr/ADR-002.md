# ADR-002: Sparse-Coverage Rendering for Offal Cuts

**Status: Superseded** — see ADR-004. Sparse-coverage logic was retired in PR 3
(commit after 294b214). Offal cuts now live at `/offal`. The main tool renders all
canonicals uniformly. Cross-reference ADR-004 for the two-product split decision.

---

**Date:** 2026-06-07
**Original Status:** Active (at time of drafting)

## Decision

Offal canonicals carried `coverage: "sparse"` in `data/canonicalCuts.ts`. The
`app/cuts/[canonical]/page.tsx` template suppressed "Countries that use it" and "Names by
country" sections when `cutRow.coverage === "sparse"`. A render-time safety net additionally
filtered any `namesByCountry` row whose name (case-insensitive, trimmed) exactly equalled the
canonical's title-cased ID string — catching silent English-fallback strings on any cut,
regardless of its coverage flag.

## Rationale

Offal cuts had sparse researched regional vocabulary by design. Only the countries where a cut
has a distinct retail term got an entry (e.g. tongue: USA "Beef tongue", Mexico "Lengua", France
"Langue de boeuf", UK "Ox tongue"). For the remaining ~110 regions no researched data existed —
rendering their canonical English name as a "translation" would assert false coverage and harm
dataset credibility. Suppression was the honest choice within the main tool's single-product
architecture.

## Why Superseded

The sparse-coverage pattern was a symptom of fitting offal into a data model keyed on muscle
cuts × country pairs. Offal cuts do not fit that model structurally: the interesting query is
cultural-prep traditions (Argentine achuras, Mexican lengua, Nalli Nihari), not bilateral name
comparison. PR 3 separated offal into its own product at `/offal`, removing the architectural
friction that necessitated the suppression flags. The main tool now renders all canonicals
uniformly. Offal canonicals remain in the `CanonicalId` union and are reachable at
`/cuts/[canonical]` with a cross-link to their dedicated `/offal/cuts/[id]` page.

The `CoverageDensity` type and all `coverage: "sparse"` fields were removed from the codebase
in the same PR. The safety-net heuristic (filtering fallback English names) was retained as it
is useful for all canonicals, not just offal.
